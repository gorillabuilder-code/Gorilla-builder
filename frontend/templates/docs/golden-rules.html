Drafting Your "Golden Rules"

The ultimate customization engine. Set strict, unbreakable boundaries for the AI Swarm.

Beyond the Presets

While Visuals, Frameworks, and Code Style cover the basics, every developer has highly specific quirks. Maybe you absolutely despise the fetch() API and demand the agent use axios. Maybe your brand guidelines dictate that every application you ever build must strictly be in Dark Mode. Maybe you want the AI to never, ever use browser alert() popups for error handling.

You define these boundaries using Golden Rules.

The Interactive Setup

In Step 5 of the Agent Skills wizard, you don't just fill out a text box—you engage in an interactive chat with a specialized setup agent.

  1. Brainstorming: You chat naturally with the agent, explaining your preferences (e.g., "I want you to always use dark backgrounds, never use alert boxes, and make sure all buttons have a rounded-full pill shape.").
  2. Refinement: The agent will help you refine these into strict, executable rules.
  3. The JSON Extraction: Once you tell the agent "I'm done," the agent autonomously extracts your conversation, formats it into a highly optimized markdown list, and outputs a hidden JSON payload.
// Under the hood, the AI generates this payload to save to your Supabase profile:
{
  "final_rules": "- Always enforce Dark Mode styling globally.\n- Never use window.alert(); use Shadcn Toast components for errors instead.\n- Force all buttons to use 'rounded-full' utility classes."
}

How Golden Rules are Enforced

Once saved, your Golden Rules are embedded directly into the core memory of the Gor://a orchestration engine.

When the Planner maps out your app, it reads your rules to ensure the architecture doesn't violate them. More importantly, when the Coder executes the plan, your Golden Rules are appended to the very bottom of its System Prompt as a "CRITICAL OVERRIDE."

Use Sparingly: Golden Rules are absolute commands. If you write conflicting rules (e.g., "Always use Vanilla CSS" but "Always use Tailwind classes for buttons"), the AI will suffer from cognitive dissonance, potentially causing formatting errors. Keep your rules clear, concise, and structural.

Examples of Great Golden Rules

  • "Always wrap API calls in a try/catch block and log errors to console.error."
  • "Never generate dummy data or placeholders; leave arrays empty if no data is provided."
  • "Always build mobile-first responsive layouts using Tailwind 'md:' and 'lg:' prefixes."
  • "Never create additional routing files; keep all Express API endpoints inside server.js."
