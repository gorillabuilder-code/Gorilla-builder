Masterclass: Writing the Perfect Prompt

Garbage in, garbage out. Learn how to speak to the Gor://a Swarm to generate production-ready, full-stack infrastructure.

The "VybeCoding" Trap

Most AI coding assistants have trained developers to be lazy. Users type "make a cool chatbot app" and expect a production-ready SaaS. On generic platforms, this results in a fragmented, single-file hallucination that breaks the moment you try to add a database.

Gor://a Builder is a 25,000-line orchestration engine. It controls a live Node.js/Express backend and a React/Tailwind frontend. To unlock its full power, you need to stop prompting like a user, and start prompting like a Product Manager delegating to a Senior Full-Stack Engineer.

The Anatomy of a Perfect Prompt

A golden prompt for Gor://a Builder contains three core elements:

  1. The Frontend Vision: Layout, framework, and visual hierarchy.
  2. The Backend Infrastructure: API routes, data logic, and integrations.
  3. The Data Layer: Supabase persistence and schema requirements.

The Golden Example:

"Build a SaaS dashboard for tracking SEO metrics. Create a React UI with a dark-mode sidebar and a main content area displaying three metric cards and a line chart placeholder. On the backend, create an Express.js route in routes/seo.js that returns mock JSON data for the chart. Wire the frontend to fetch from this API on load. Ensure the UI uses Tailwind glassmorphism."

Directing the Backend (Express.js & API)

Gor://a natively runs a headless Node.js server. If you want backend logic, explicitly ask for it. The Swarm knows your entry point is server.js.

  • Do: "Create an Express route at /api/waitlist that accepts a POST request with an email."
  • Don't: "Make a waitlist form." (The AI might just make a frontend form that does nothing when clicked).

Triggering the Unified Gateway

Gor://a's massive technical moat is its Unified AI & Auth Gateway. You can trigger complex SaaS features simply by asking for them in plain English. The agent knows how to use your gb_live_ API key.

  • Authentication: "Add a login page and use the Gor://a Auth gateway for Google Sign-in."
  • AI Integration: "Add a chat interface that sends the user's message to the /api/v1/chat/completions gateway and streams the response back."
  • Image Generation: "Add a button that calls the /api/v1/images/generations gateway to generate an image based on the input field."

Token Economy Tip: Be explicit but concise. If you write a 500-word essay for your prompt, the Planner agent has to process massive amounts of text, which consumes your monthly tokens faster. Keep it structured and bulleted.
