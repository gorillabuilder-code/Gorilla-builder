Exporting to ZIP

The death of vendor lock-in. Take your code and run.

Your Code Belongs to You

The ultimate trap of closed-ecosystem app builders is vendor lock-in. You build a great product, hit a scaling limit, and realize you cannot export your code. You are trapped paying their premium hosting fees forever.

Gor://a Builder fundamentally rejects this model. We are an orchestration engine, not a prison. Every line of code generated by the Swarm is standard React, Node.js, and Tailwind. It is designed to be ejected at any time.

How to Export

If you are on the Premium (Pro) Tier or Enterprise Tier, you have unlimited access to source code exports.

  1. Open your project in the Editor.
  2. Look at the top right of the topbar header.
  3. Click the golden ZIP button.

The Gor://a backend will instantly query the Supabase Virtual File System, compress your entire project directory, and serve it to your browser as a clean .zip file.

Running Your App Locally

Once downloaded, unzipping the project gives you a standard full-stack environment. Because we use the Golden Boilerplate, running it locally is exactly what you are used to as a developer:

# 1. Unzip the project and navigate into the folder
cd gorilla_project_abc123

# 2. Install the exact dependencies the AI defined
npm install

# 3. Create an .env file
# (Copy your VITE_SUPABASE_URL and GORILLA_API_KEY into it)

# 4. Boot the Vite and Express servers
npm run dev

Environment Variables: For security reasons, we do not export your raw .env file into the ZIP archive. You will need to manually create an .env file on your local machine and copy your keys from the Gor://a settings panel to ensure your app can still hit the Unified Gateway and your Supabase database.
