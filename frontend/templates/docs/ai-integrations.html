LLM Chat Integration

Inject world-class AI models directly into your applications without managing OpenAI or Anthropic billing accounts.

The API Key Dilemma

Building an AI wrapper or SaaS requires API keys. Normally, you have to create an OpenAI account, load a credit card, manage rate limits, and secure your keys on your backend to prevent abuse. For an indie developer trying to validate a weekend project, this is a massive barrier to entry.

Gor://a Builder bypasses this by allowing your generated applications to piggyback off your Gor://a API Tokens via the Unified AI Gateway.

How the Gateway Works

Every Gor://a workspace is injected with a process.env.GORILLA_API_KEY. This key is securely held in your Node.js backend (it is never exposed to the React frontend). When you prompt the Swarm to build an AI feature, it routes the traffic through our proxy.

  1. Your React frontend sends the user's input to your Node.js backend.
  2. Your Express route forwards the request to https://app.gorillabuilder.dev/v1/chat/completions using your gb_live_ key.
  3. Our gateway authenticates you, securely proxies the request to OpenRouter (using high-performance models like LLaMA 3.2 or DeepSeek), and streams the response back to your app.

How the Swarm Writes It

The Coder Agent is strictly instructed to use standard OpenAI-compatible JSON payloads, ensuring maximum compatibility. It writes highly optimized Express controllers like this:

// In your Express backend (routes/ai.js)
import express from 'express';
const router = express.Router();

router.post('/chat', async (req, res) => {
  try {
    const response = await fetch('https://app.gorillabuilder.dev/v1/chat/completions', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.GORILLA_API_KEY}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        messages: req.body.messages,
        // The Gateway handles model routing and load balancing automatically!
      })
    });
    
    const data = await response.json();
    res.json(data);
  } catch (error) {
    res.status(500).json({ error: "Failed to connect to AI Gateway" });
  }
});

export default router;

Token Economics: When your generated app uses the LLM integration, it burns tokens from your main Gor://a account balance. The Gateway charges a highly optimized rate (roughly 0.3 to 0.5 credits per 1 actual API token) depending on the model used. Because you start with 500,000 free tokens, you have more than enough runway to build and test your SaaS before upgrading to Premium.

Prompting for AI Features

To build an AI app, simply tell the Swarm what the AI should do. The orchestration engine handles the boilerplate.

Example Prompt: "Build a 'Roast My Landing Page' app. The user types their URL or pitch, and the Express backend calls the Gor://a chat completions gateway to generate a funny, sarcastic roast. Display the result on the frontend with a typing animation."
