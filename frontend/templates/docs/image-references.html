Using Image References

Show, don't just tell. Pass UI screenshots directly into the Swarm's vision pipeline.

How Vision Integration Works

Sometimes describing a complex dashboard layout in text is impossible. Gor://a Builder allows you to upload an image directly into the Dashboard prompt bar using the icon.

When you upload an image, the Gor://a engine compresses it and safely saves it to your Virtual File System as a Base64 string. When the Planner and Coder agents boot up, they utilize state-of-the-art vision models to "see" your image and translate the visual layout, color palette, and spacing directly into Tailwind CSS classes.

The Prompt + Image Combo

An image alone is usually not enough to build a fully functional app. You must combine the image with a structural text prompt to get the best results.

// Good Text Prompt to accompany an image:
"Replicate the UI in the attached image using Tailwind CSS and React. 
Make sure the sidebar is fully functional and mobile-responsive. 
Wire the 'Submit' button seen in the image to a new Express backend route."

Best Practices for Image Uploads

To avoid token waste and hallucinated code, follow these guidelines when uploading image references:

  • Crop to the Component: Don't upload a 4K screenshot of your entire desktop browser if you just want the AI to build a specific pricing card. Crop the image to exactly what you want built.
  • High Contrast is Better: Vision models struggle with low-contrast, blurry, or highly compressed images. Ensure your screenshot is crisp.
  • Avoid Massive Text Blocks: If your image contains paragraphs of text, the AI will waste tokens trying to transcribe the text rather than focusing on the layout.

Mid-Chat Image Overrides: Did you know you can upload images mid-session? Once your WebContainer is running, you can click the image icon in the Editor's chat bar to upload a new reference. The engine will safely overwrite the old image in your database, allowing you to visually guide the AI through iterations.
