Linking Your GitHub Account

Granting Gor://a secure access to create and manage your private repositories.

Why GitHub?

Vercel, Netlify, and other modern hosting platforms rely heavily on Git integration. By pushing your code to GitHub first, you unlock Continuous Integration / Continuous Deployment (CI/CD). This means that after your initial launch, any future changes you push from Gor://a will automatically trigger Vercel to rebuild and update your live site.

Step 1 of the Deployment Wizard

If you have not already linked your GitHub account via the Settings page, Step 1 of the Deployment Wizard will prompt you to do so.

  1. Click Connect Account.
  2. You will be securely redirected to GitHub.
  3. Gor://a will request the user:email and repo scopes.
  4. Upon authorization, you will be redirected back to the Deployment Wizard, and Step 1 will be marked as "Completed."

Security Note: Gor://a Builder strictly uses this OAuth token to create new, private repositories for your generated apps and to push code to them. We do not read, clone, or modify any of your pre-existing manual repositories.

What if my primary email is hidden?

Many developers hide their primary email on GitHub for privacy. Gor://a's backend is specifically engineered to handle this. If your email is fully hidden, our backend will autonomously construct a proxy email (e.g., username@noreply.github.com) so that your authentication flow succeeds without breaking your privacy settings.
