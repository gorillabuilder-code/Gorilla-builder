Preparing for Deployment

Taking your application out of the sandbox and into the real world.

The "It works on my machine" Problem

Traditional AI coding assistants are notorious for generating code that works perfectly in their proprietary sandbox, but immediately breaks when you try to host it on a real server. Missing environment variables, incorrect routing, and broken build scripts are the bane of indie developers.

Gor://a Builder solves this by employing a strict 4-Step Deployment Pipeline. We do not just hand you the code; we autonomously restructure it for edge hosting, encrypt it into a GitHub repository, and inject the necessary deployment URLs.

The 4-Step Pipeline Overview

When you click "Deploy App" from the Editor, you enter the Deployment Wizard. This wizard handles:

  1. Linking GitHub: Establishing a secure OAuth connection to your GitHub account to create a private repository.
  2. Vercel Optimization: Pausing the WebContainer to allow the Swarm to restructure your Node.js Express server into Vercel Serverless Functions.
  3. Pushing the Code: Converting your Virtual File System into a Git Tree and committing it to GitHub.
  4. The Vercel Handoff: Formatting a one-click deployment URL that injects your Supabase and Gor://a API keys directly into Vercel's environment settings.

Premium Tier Required: Because automated GitHub deployment utilizes advanced AI refactoring and extensive backend bandwidth, this feature is restricted to users on the Premium (Pro) and Enterprise tiers.
