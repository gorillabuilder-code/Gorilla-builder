Speech-to-Text (Whisper)

Build voice-enabled applications using state-of-the-art audio transcription.

Voice as an Interface

Typing is slow. If you are building note-taking apps, AI therapists, or language learning tools, voice input is a mandatory feature. Gor://a Builder natively integrates the Whisper-v3-turbo model via Fireworks AI into the Unified Gateway.

The STT Pipeline

Handling audio requires a slightly more complex full-stack architecture, but the Swarm knows exactly how to build it:

  1. Frontend Capture: The React frontend uses the native browser MediaRecorder API to capture microphone input and compile it into an audio Blob.
  2. Multipart Form Post: The Blob is sent to your Express backend via FormData.
  3. Gateway Forwarding: Your backend forwards the file to /api/v1/audio/transcriptions using your GORILLA_API_KEY.
  4. Transcription: The Gateway transcodes the audio and returns the highly accurate text string.

What about Text-to-Speech (TTS)?
To protect your token balance, the Gor://a Coder Agent is strictly instructed never to use expensive cloud APIs for Text-to-Speech. Instead, it utilizes the browser's native window.speechSynthesis Web Speech API. This provides instant, high-quality voice playback on the frontend for absolutely zero tokens.

Token Cost & Heuristics

Audio transcription is billed based on duration. Because audio formats vary wildly in compression, the Gor://a Gateway uses a heuristic file-size calculation (assuming ~1MB per minute of standard m4a/mp3 audio). You are billed exactly 100 tokens per estimated minute of transcribed audio.

How to Prompt for Voice Features

Example Prompt: "Build an AI Voice Journal. The user clicks a microphone icon to start speaking. When they stop, send the audio to the Gor://a audio transcription gateway. Take the transcribed text, pass it to the chat completions gateway to summarize it into 3 bullet points, and then use the browser's native speech synthesis to read the summary back to them out loud."
