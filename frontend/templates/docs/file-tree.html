Understanding the File Tree

Demystifying the "Golden Boilerplate" and how your Full-Stack app is structured.

No Black Boxes

Most "VybeCoding" platforms hide your code in a proprietary black box. Gor://a Builder gives you a standard, ejectable, industry-standard file structure known as the Golden Boilerplate.

When you create a project, this structure is instantly provisioned in your Explorer.

The Golden Structure

/
├── src/                  # The Frontend Realm (React + Vite)
│   ├── components/       # Reusable UI parts
│   │   └── ui/           # Pre-installed Shadcn components
│   ├── pages/            # Page-level components (e.g., Index.tsx)
│   ├── lib/              # Utility functions (utils.ts)
│   ├── App.tsx           # React Router and main layout
│   ├── main.tsx          # React DOM entry point
│   └── index.css         # Tailwind directives
├── routes/               # The Backend Realm (Express.js)
│   └── api.js            # Custom REST API endpoints
├── migrations/           # Database Realm (SQL)
│   └── 001_init.sql      # Supabase schema definitions
├── server.js             # The Node.js Backend Entry Point
├── package.json          # Dependencies & Scripts
├── tailwind.config.js    # Tailwind theme rules
└── vite.config.ts        # Bundler settings

Where the Magic Happens

  • server.js: This is your Node.js backend. It serves your Express API routes and handles the Vercel serverless translation.
  • routes/: The AI will dynamically create files here (e.g., auth.js, stripe.js) and mount them into server.js to keep your backend modular.
  • src/components/ui/: Gor://a comes pre-loaded with Shadcn. The AI knows how to import these accessible components instantly without running slow NPX scripts.

The Shark Filter: Notice anything missing? You won't see node_modules, package-lock.json, or .git folders in the Explorer. We deliberately filter these out of the UI and the AI's context window. They are massive, irrelevant files that would clutter your screen and burn through your token limits in seconds.
