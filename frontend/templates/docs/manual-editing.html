Manual Code Editing

Sometimes you don't need an AI to change a hex code. Take full control with the built-in IDE.

You Are Always in Control

Gor://a Builder is an AI-first orchestration engine, but we are developers too. We know that spending 500 tokens and waiting 20 seconds for an AI to change p-4 to p-6 is a miserable experience.

That is why every Gor://a workspace includes a fully functional, syntax-highlighted CodeMirror 6 editor.

How to Edit Manually

  1. In the Editor interface, look at the Explorer (File Tree) on the left.
  2. Click on any file (e.g., src/pages/Index.tsx).
  3. Click the Code tab at the top of the main stage.
  4. Type your changes directly into the editor.

Deterministic Syncing

We built the editor to be completely frictionless. There is no "Save" button.

  • Auto-Save: When you stop typing for 800 milliseconds, the editor automatically fires a payload to the backend, safely upserting your changes into the Supabase Virtual File System.
  • Live Hot-Reload: Simultaneously, the changes are written directly into the WebContainer's memory (webRunner.writeFile()). Vite detects the file change and instantly hot-reloads your preview iframe.

The Safety Lock: If the AI Swarm is currently executing a plan (indicated by the spinning agent icon), manual file saving is temporarily queued. Once the AI finishes its commit, your manual pending changes are applied chronologically to prevent file-state corruption.

Handling Images

If you click on an image asset or a .b64 file in the Explorer, the CodeMirror editor will automatically hide itself and render a visual image preview instead. No more staring at endless walls of Base64 gibberish.
