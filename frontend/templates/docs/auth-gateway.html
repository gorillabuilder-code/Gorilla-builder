The Gor://a Auth Gateway

Add Google and GitHub login to your generated apps instantly. No cloud consoles, no API keys, zero configuration.

The "OAuth Trap" in Standard AI Tools

If you've ever tried to ask a generic AI coding assistant to "add Google Login," you know the nightmare that follows. The AI writes the code, but then halts and tells you: "Now go to the Google Cloud Console, create a new project, configure the OAuth consent screen, whitelist your localhost URIs, generate a Client ID and Secret, and paste them into an .env file."

This process takes hours, breaks constantly across different environments, and completely ruins the flow of rapid prototyping.

The Gor://a Solution

Gor://a Builder eliminates this trap with the Unified Auth Gateway. Because your app is running inside our orchestrated WebContainer, we can securely proxy authentication requests through our own pre-configured, enterprise-grade OAuth applications.

When you prompt the Swarm: "Add a login page with Google and GitHub," the Coder Agent doesn't write raw OAuth logic. Instead, it utilizes the VITE_GORILLA_AUTH_ID injected into your environment and points your users to our Hosted Login Page.

How the Swarm Implements Auth

The Coder Agent automatically generates a utility file (usually src/lib/auth.ts) that interfaces with our Gateway. It looks something like this:

// Automatically generated by the Swarm
export const login = (provider) => {
  // provider can be 'google' or 'github'
  const authId = import.meta.env.VITE_GORILLA_AUTH_ID;
  const returnUrl = encodeURIComponent(window.location.origin);
  
  // Redirects to the secure Gor://a Hosted Auth Gateway
  window.location.href = `https://app.gorillabuilder.dev/api/v1/app-auth/login?auth_id=${authId}&return_url=${returnUrl}`;
};

// Listen for the user payload returning from the Gateway
export const onAuthStateChanged = (callback) => {
  const user = localStorage.getItem('gorilla_user');
  callback(user ? JSON.parse(user) : null);
};

Seamless UX: When your users click "Login," they are briefly routed through our secure gateway, authenticated via Google or GitHub, and instantly bounced back to your running application with their user profile (Name, Email, Avatar) securely stored in state. It works flawlessly on both the browser IDE and your live Vercel deployment.

Best Practices for Prompting

  • Keep it simple: Just say, "Protect the dashboard page so only logged-in users can see it. Add Google/GitHub login." The AI handles the rest.
  • Database Linking: If you are also using the Supabase integration, you can say, "When a user logs in, save their email and name to the Supabase users table." The Coder Agent will automatically wire the Gateway's response to your Express backend.
