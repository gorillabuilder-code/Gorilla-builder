Prompting Guide

The quality of your app depends heavily on how you describe it. Think of the AI as a junior developer: it knows how to code perfectly, but it needs clear instructions on what to build.

The Golden Rule: Be Specific

Vague prompts lead to generic apps. Specific prompts lead to polished products.

❌ Bad Prompt

"Make a store."

✅ Good Prompt

"Create a modern e-commerce store for selling sneakers. It should have a dark theme, a grid of products with hover effects, a shopping cart sidebar, and a checkout form."

Structure of a Great Prompt

  1. Role/Context: "Act as an expert React developer..."
  2. Core Functionality: "Build a task manager with drag-and-drop..."
  3. Design Style: "Use a minimalist black and white aesthetic..."
  4. Technical Constraints: "Use Tailwind CSS and keep everything in one file if possible."

Iterative Prompting

You don't have to get it right on the first try. Start simple, then use the Chat Interface to refine:

  • "Now add a search bar."
  • "Make the buttons rounder."
  • "Change the background color to dark blue."
