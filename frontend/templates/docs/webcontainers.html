How WebContainers Work

Running a live, headless Node.js backend directly inside your browser tab.

The Legacy Cloud Trap

Most AI coding platforms use one of two broken models to run your code:

  1. Static Only: They only generate HTML/CSS/React and mock the data, leaving you stranded when you actually need a database or user authentication.
  2. Cloud Compute: They spin up expensive, sluggish AWS/GCP Docker containers for every user, which introduces massive latency and forces them to charge you $40/month just to cover server costs.

The Gor://a Solution: WebContainers

Gor://a Builder bypasses the cloud entirely. We utilize WebContainer technology to boot a micro-operating system directly inside your browser's memory. This means you have a native Node.js runtime executing locally on your machine, orchestrated by our remote Python agents.

The Boot Sequence

When the Coder agent finishes its blueprint, the webrunner.js orchestrator takes over:

  1. Mounting: It pulls your codebase from the Supabase Virtual File System (VFS) and mounts it into the browser's memory.
  2. Dependency Resolution (pnpm install): It scans the package.json the AI generated and downloads dependencies at lightning speed directly from the NPM registry.
  3. The Dev Server (npm run dev): It boots up Vite and Express simultaneously.
  4. Live Preview: It binds to a local, credentialless URL and displays the live app in the Editor's iframe.

Cross-Origin Security: To run a full operating system in a browser tab, Gor://a uses strict Cross-Origin-Embedder-Policy: credentialless headers. This ensures that your local environment is highly secure and perfectly isolated from the rest of the web.

Why This is a Game Changer for Indies

Because the compute happens on your device, Gor://a doesn't pay for idle servers. This architectural breakthrough is exactly how we can afford to give you 500,000 free tokens and full-stack capabilities without pricing you out of the market. Zero latency, zero cloud costs, infinite possibilities.
