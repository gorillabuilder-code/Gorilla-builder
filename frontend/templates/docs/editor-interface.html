The Editor Interface

Your unified command center. We killed the terminal, the code editor, and the browser tabs, and merged them into one seamless workspace.

Beyond the Chat Wrapper

The "VybeCoding" trap usually forces you into a split-screen nightmare: a chat window on the left, and a fragmented, un-runnable code snippet on the right. Gor://a Builder replaces this with a true, professional-grade In-Browser IDE tailored for AI-first development.

The Four Zones of the IDE

1. The Swarm Console (Left Sidebar)

This is where you communicate with your agents. Unlike standard AI tools, this isn't just a chat box—it's a live timeline of multi-agent orchestration.

  • The Thought Timeline: Watch the Planner, Reasoner, and Coder communicate via MCP (Machine Communication Protocol). You can see exactly which files are being analyzed and updated in real-time.
  • The Progress HUD: Live indicators showing the current phase (Planner, Coder, Files) and the WebContainer boot status.
  • The Input Matrix: Submit text prompts, upload UI reference images mid-chat, or hit the "Xmode" button for deep architectural refactoring.

2. The Explorer (File Tree)

A live view of your Virtual File System (VFS). As the AI writes your React components and Express routes, they instantly appear here. You can click any file to open it in the manual editor, or toggle folders open and closed.

3. The Stage (Code vs. Preview)

The center of the screen is your primary workspace. Using the top tabs, you can instantly toggle between:

  • Preview Mode: A live, interactive iframe of your running React + Node.js application.
  • Code Mode: A robust CodeMirror 6 instance with full syntax highlighting for inspecting or manually tweaking the AI's output.

4. The Topbar (Global Actions)

The persistent header gives you one-click access to critical functions:

  • Token HUD: Live updates on your token burn rate.
  • Deployment Pipelines: Quick-launch buttons to push your code to GitHub or deploy directly to Vercel.
  • ZIP Export: Download your entire codebase locally. No vendor lock-in, ever.

Frictionless Iteration: Because the IDE is wired directly to the WebContainer, asking the AI to "change the button to red" updates the file in the VFS, which hot-reloads the preview iframe in milliseconds. You never have to touch a save button.
