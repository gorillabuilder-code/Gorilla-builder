Account & Settings

Your settings page is where you connect external services to give your AI agent its superpowers. Gor://a Builder is powerful on its own, but it becomes an unstoppable "Zero-to-Production" OS when you link your existing developer ecosystem.

Linking External Accounts (OAuth)

To use advanced features like autonomous database creation, one-click deployments, or Figma conversions, you must grant Gor://a Builder secure OAuth access to the following platforms:

  • GitHub: Required for the Deployment Pipeline. Gor://a will use this token to create private repositories on your behalf, push optimized vercel.json code, and trigger live Vercel deployments. We only request the strict scopes necessary to manage repositories generated by Gor://a.
  • Supabase: Required for Full-Stack databases. By linking the Supabase Management API, Gor://a can autonomously provision new PostgreSQL databases, fetch your environment keys, inject them into your WebContainer's .env file, and execute raw SQL migrations automatically.
  • Figma: Required for the UI Compiler. Allows Gor://a's AgentSkills to read specific design nodes, extract layer hierarchies, and download image assets to compile pixel-perfect React/Tailwind code.

Seamless Workflow: You only need to link these accounts once. After linking, the AI handles the provisioning, committing, and compiling entirely in the background. No more hunting for cloud secrets!

Your Unified Gateway Key (`gb_live_`)

Every user is assigned a unique, secure API key that begins with gb_live_. This is your master key for the Gor://a AI & Auth Proxy Gateway.

When the AI builds complex features for you—such as an in-app AI Chatbot, Fireworks Image Generation, or Whisper Speech-to-Text—it automatically injects this key into your Node.js backend environment variables. This allows your generated app to securely make AI calls through our infrastructure without you needing to provide your own expensive OpenAI or Anthropic keys.

// Example of how your generated Node.js backend uses the Gateway:
const response = await fetch('https://api.gorillabuilder.dev/v1/chat/completions', {
  headers: {
    'Authorization': `Bearer ${process.env.GORILLA_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ messages: userChatHistory })
});

Regenerating Your Key

If you suspect your API key has been exposed (for example, if you accidentally committed your .env file to a public GitHub repository outside of Gor://a's secure pipeline), you must revoke it immediately to prevent token drain.

Navigate to Settings and click the Regenerate API Key button. This will instantly invalidate your old key across our entire proxy network and generate a new one. Note: You will need to prompt the agent to update the .env file in your existing projects with the new key so your apps continue to function.

Data Privacy & Deletion

We respect your data and your code. Your source code lives in your browser's WebContainer and our secure Virtual File System. If you wish to wipe your data, deleting a project from the dashboard instantly and permanently purges all associated files and chat histories from our servers.
