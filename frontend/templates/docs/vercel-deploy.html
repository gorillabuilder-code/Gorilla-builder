Deploying to Vercel

The final step. Launching your application to the edge with zero-config security.

Step 4: Ready for Liftoff

If you have made it to Step 4, your application's source code is safely encrypted in a private GitHub repository. You are now exactly one click away from a live, production URL.

Zero-Config Security

The hardest part of deploying an app is managing environment variables. If you deploy your code but forget to copy your Supabase keys into Vercel, your app will crash.

Gor://a Builder solves this with Zero-Config Security. When generating the deployment link, Gor://a automatically fetches your GORILLA_API_KEY, your VITE_SUPABASE_URL, and your VITE_SUPABASE_ANON_KEY from the backend. It injects these keys directly into the URL parameter payload.

The One-Click Deploy

When you click Deploy to Vercel, you are taken to the official Vercel Import dashboard.

  1. Vercel will ask you to authorize access to the private GitHub repository Gor://a just created for you.
  2. Because of the payload Gor://a attached to the URL, Vercel will automatically populate your Environment Variables. You do not need to paste anything.
  3. Click Deploy.

Vercel will install your dependencies, run the Vite build script, and deploy your Serverless Node.js functions. Within 60 seconds, you will be handed a live, SSL-secured .vercel.app domain.

Custom Domains: Once deployed, you can log into your Vercel dashboard to attach a custom domain (e.g., myawesomeapp.com) to your project for free.
