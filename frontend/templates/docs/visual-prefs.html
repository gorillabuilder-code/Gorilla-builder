Visual Preferences

Dictate how your agent handles iconography and graphical elements in your UI.

The Impact of Iconography

Icons define the "feel" of a web application. A SaaS dashboard requires crisp, uniform SVG icons, while a fun, consumer-facing social app might benefit from the playful, native feel of text-based emojis. Gor://a Builder allows you to enforce this globally.

Option A: Clean SVGs (The Production Standard)

When you select Clean SVGs, the Coder agent is explicitly instructed to avoid text emojis entirely. Instead, it will leverage professional SVG icon libraries.

Because Gor://a's Golden Boilerplate comes pre-configured with modern tooling, the agent will naturally utilize lucide-react or @phosphor-icons/web to generate beautiful, scalable vector graphics.

// Example Output: Clean SVGs
import { Settings, User, LogOut } from 'lucide-react';

export default function Sidebar() {
  return (
    <nav className="flex flex-col gap-4">
      <a href="/profile"><User size={18} /> Profile</a>
      <a href="/settings"><Settings size={18} /> Settings</a>
    </nav>
  );
}

Option B: Emojis (The Indie Hacker Vibe)

Sometimes you just want to ship fast and keep the bundle size small. When you select Emojis, the agent is instructed to use native unicode emojis for all visual representations. This is perfect for rapid prototyping, landing pages, and "vybe" coding.

// Example Output: Emojis
export default function Sidebar() {
  return (
    <nav className="flex flex-col gap-4">
      <a href="/profile">👤 Profile</a>
      <a href="/settings">⚙️ Settings</a>
    </nav>
  );
}

Design Tip: If you are importing designs via the Figma Compiler, the AI will naturally attempt to preserve the icons used in your Figma file, regardless of this setting. However, this setting will still apply when you prompt the agent to create entirely new components from scratch.
