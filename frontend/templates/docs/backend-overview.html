The Node.js + Express Backend

Escaping the "Frontend-Only" trap. How Gor://a Builder orchestrates real, live backend infrastructure in your browser.

The VybeCoding Illusion

Most AI coding tools on the market today are fundamentally crippled: they can only build static frontends. If you ask them for a database or a backend route, they either mock the data using localStorage or instruct you to manually set up a server, manage CORS, and juggle environment variables. They leave you stranded when it comes to actual infrastructure.

Gor://a Builder was engineered to kill backend boilerplate. Every project you create is a true full-stack application, running a headless Node.js environment directly inside the WebContainer alongside your React frontend.

The Golden Boilerplate: Backend Architecture

When the Gor://a Swarm spins up a new project, it inherently understands our Golden Boilerplate backend structure. It uses modern ES Modules (import/export) and Express.js to build your API.

/
├── server.js             # The main Express application entry point
├── routes/               # Modular API routes
│   ├── api.js            # General API endpoints
│   ├── auth.js           # Authentication endpoints (via Unified Gateway)
│   └── stripe.js         # Webhook handlers
└── .env                  # Securely injected environment variables

How the Swarm Writes Backend Code

Because the AI understands this architecture, it doesn't dump thousands of lines of spaghetti code into a single file.

When you prompt, "Add an email waitlist," the Planner Agent will instruct the Coder Agent to:

  1. Create a new file at routes/waitlist.js to handle the POST request.
  2. Update server.js to import and mount the new route (app.use('/api/waitlist', waitlistRouter)).
  3. Update the React frontend to fetch data from /api/waitlist.

The Unified Gateway Injection: If your backend requires AI LLM generation, image processing, or authentication, the agent writes routes that securely communicate with the Gor://a AI & Auth Proxy Gateway. It utilizes your process.env.GORILLA_API_KEY, meaning you never have to expose your own OpenAI keys or set up complex third-party SDKs.

Seamless Frontend/Backend Communication

You never have to worry about CORS (Cross-Origin Resource Sharing) errors. The WebContainer environment uses Vite's proxy capabilities. When your React app makes a fetch('/api/data') request, Vite natively routes it to your running Express server on port 3000. It works locally exactly as it will when you deploy to Vercel.
